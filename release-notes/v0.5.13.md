## What's fixed

- **More accurate cable endpoint detection.** The CC endpoint (used by MagSafe ports) is now only mapped when it appears in the transport type field, not in the component name field. This prevents a future cable or device with "CC" in its name from being misidentified as a cable e-marker. Thanks to @0x687931 for spotting this.

## Install

`brew upgrade --cask whatcable`, or download `WhatCable.zip` from this release page and drag the new `.app` into `/Applications`.
